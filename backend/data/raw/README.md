# GemBlast Player Events: Dataset Schema

Synthetic event stream for a fictional match-3 mobile game.  
Generated by `backend/data/generate_events.py`.

---

## File

`player_events.csv` — one row per event, sorted by `event_time`.

---

## Schema

| Column | Type | Nullable | Description |
|---|---|---|---|
| `event_id` | string (UUID) | no | Unique identifier for this event |
| `player_id` | string | no | Unique player identifier (`p_00001` … `p_05000`) |
| `cohort` | string | no | Spend segment: `whale`, `mid`, or `f2p` (see below) |
| `install_date` | date (`YYYY-MM-DD`) | no | Date the player first launched the game |
| `days_since_install` | int | no | Days elapsed since install (0 = install day) |
| `event_type` | string | no | One of `session_start`, `level_complete`, `purchase`, `ad_watched` |
| `event_time` | datetime (`YYYY-MM-DD HH:MM:SS`) | no | UTC timestamp of the event |
| `session_id` | string (UUID) | no | Groups all events that belong to the same play session |
| `level_id` | int | yes | Cumulative level number the player just completed. `NULL` for all non-`level_complete` events |
| `amount_usd` | float | yes | IAP value in USD. `NULL` for all non-`purchase` events |

---

## Event Types

| Event | Description |
|---|---|
| `session_start` | Player opens the app. Always the first event in a session. |
| `level_complete` | Player successfully finishes a level. `level_id` increments globally per player. |
| `ad_watched` | Player watches a rewarded video ad (placed after level completions). |
| `purchase` | Player completes an in-app purchase. `amount_usd` is populated. |

**Session structure:** `session_start` → [`level_complete` → (optional) `ad_watched`] × N → (optional) `purchase`

---

## Player Cohorts

Players are assigned to a cohort at install time and retain that label permanently.

| Cohort | Share | Spend profile | Retention (D1 / D7 / D30) | Sessions/day |
|---|---|---|---|---|
| `whale` | 5 % | IAP-heavy; $0.99–$99.99, skewed high | ~80 % / ~57 % / ~40 % | ~3.5 |
| `mid` | 20 % | Occasional small IAPs ($0.99–$19.99) | ~64 % / ~36 % / ~22 % | ~2.0 |
| `f2p` | 75 % | Rarely or never pays; monetises via ads | ~46 % / ~17 % / ~8 % | ~1.2 |

---

## Retention Model

Daily activity follows a **power-law decay**:

```
P(active on day d) = base × (d + 1)^(−α)
```

| Cohort | base | α |
|---|---|---|
| whale | 0.95 | 0.25 |
| mid | 0.88 | 0.45 |
| f2p | 0.75 | 0.70 |

Day 0 (install day) is always active.

---

## Generation Parameters

| Parameter | Value |
|---|---|
| Players | 5 000 |
| Simulation window | 180 days (2024-01-01 → 2024-06-28) |
| Install spread | Days 0–149 (uniform) |
| Random seed | 42 |
